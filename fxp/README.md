# fxp/ — fixed-point primitives for Falcon

Python reference implementation of the fixed-point arithmetic stack for
Falcon key-generation (`ffLDL*`, `normalize_tree`) and signing
(`ffsampling`, `samplerz`, target construction). Used as the high-level
model for the paper *Toward a Secure Fixed-Point Implementation of the
Falcon Signature Scheme* (De Almeida Braga, Fouque, Lachguel, Prest).

> **Status**: research prototype; **not constant-time** (timing leaks via
> `_bankers_shift`/`_div_round`, samplerz rejection loop, byte-compare in
> berexp). A constant-time implementation is out of scope for this prototype.
>
> **Falcon-512 only**: `sample_preimage` asserts `sk.n == 512` on the fxp
> path; n=1024 needs numerical re-validation of the per-level precision drop
> in ffsampling.

## What's here

```
fxtypes.py                  FxR / FxC types (add, sub, mul, div, conjugate)
fft_fxp.py                  FFT/iFFT of FxR/FxC polynomials
fxp_constants_p63.py        FxR/FxC numerical constants at p=63
fxp_constants_p127.py       same at p=127
                            (twiddle tables + 1/√q, 1/q, σ_n, sigmin_n;
                             generated by scripts/generate_constants_fxp.sage)
ffldl_fxp.py                ffLDL*, LDL*, rsqrt, normalize_tree, keygen wrapper
ffsampling_fxp.py           ffsampling for signing (uses the normalized tree)
samplerz_fxp.py             fixed-point port of Falcon's Gaussian sampler
sign_tweak.py               end-to-end Sign with optional Section-5.1 tweak
                            (NTT-based t_frac extraction) and optional fxp
                            ffsampling path
```

All modules run against the reference Falcon implementation under
`falcon_ref/falcon.py` (the upstream Prest Falcon reference, with a
`SecretKey` shim for the attribute API the fxp module is built against):
we *use* its NTRU-gen, hash-to-point, base sampler tables, etc., and
*replace* only the floating-point arithmetic.

## Type system

```
FX_{m,p} = { 2^{m-p} · x : x ∈ Z, |x| < 2^p }
```

- `p`  : precision (bits of the signed mantissa)
- `m`  : magnitude bound (every value satisfies |v| < 2^m)
- `FxR`: real fixed-point with fields `(x, m, p)`
- `FxC`: complex fixed-point `re + i·im`, both components share `(m, p)`.
  Uses the **complex-modulus** convention: `m` bounds `|z|`, not each
  component separately. This keeps FFT merges to +1 bit of `m` per level
  (rather than +2).

### Arithmetic summary

| op | output `m` | output `p` | notes |
|---|---|---|---|
| `a + b` (FxR or FxC) | `a.m == b.m` | unchanged | exact, asserts shared `(m, p)` |
| `a - b` | idem | — | — |
| `a * b` (FxR) | `a.m + b.m` | unchanged | banker's shift by `p` on exact product |
| `a * b` (FxC) | `a.m + b.m` | unchanged | Lagrange-preserving, banker's shift by `p` |
| `a.div(b, m_out)` | `m_out` (caller) | unchanged | single banker's-round division |
| `a.div_int(k, m_out)` | `m_out` (caller) | unchanged | div by integer scalar (e.g. /q after NTT) |

Division does not have a natural tight `m_out` (it depends on a
context-specific lower bound on `|b|`), so the caller supplies it.

All divisions use big-int `divmod` (banker's round). They are exact and
match the reference behaviour bit-for-bit; they are **not constant-time**
(divmod's timing depends on operand size). A constant-time/masked impl
would replace `_div_round` with a Newton–Raphson divider.

## Key numerical tricks

### In ffLDL / LDL (keygen)

- **Symplectic D_11 at the NTRU root**: `det(G_root) = q²` by construction,
  so `D_11_root = q²/D_00_root` directly instead of the usual subtraction.
  Avoids catastrophic cancellation; see `ldl_fft_fxp_ntru_root`.
- **Reformulated D_11 at internal levels**: `D_11 = G_11 − adj(L_10)·G_10`
  (not `G_11 − |L_10|²·G_00`). Avoids squaring `L_10`, keeping
  intermediate `m` low.
- **Renormalization to m_D** between levels, so `m` does not drift up
  across recursion levels; Lemma 9 guarantees a uniform bound.

### In rsqrt (for `normalize_tree`)

- Newton-Raphson `y ← 0.5 · y · (3 − x·y²)` (no division in the loop).
- Pure fxp, no float anywhere: `y0_anchor` is `FxR(x=…, m=-6, p=63)`
  representing `1/√q` for Falcon `q = 12289`, hardcoded as a literal
  (precomputed offline via `inv_sqrt_int_fxr`, which uses `math.isqrt`
  — integer arithmetic only, despite the module name).
- Falcon's NTRUGen enforces `gs_norm ≤ 1.17² · q`, which guarantees
  that every ffLDL leaf D_ii lies in `[q/1.37, 1.37·q]` **pointwise**
  — verified empirically at 0 violations on 15 360 leaves. With
  `y_0 = 1/√q` and ε₀ ≤ 0.17, **6 NR iters** reach `|ε| ≪ 2^-63`
  with margin to spare (the `iters=6` default; iters=5 is borderline
  by ~½ bit in worst case). For `p = 127`, the same `iters=6` is
  adequate but the anchor must be rebuilt via
  `inv_sqrt_int_fxr(q, m=-6, p=127)`.
- Convergence radius: `|ε_0| < 2/3` (i.e. `x ∈ [(y0/3)^-2, (5y0/3)^-2]`).
  Caller is responsible for ensuring `x` is in this domain — no runtime
  float bound check.

### In Sign (Section-5.1 tweak, `sign_tweak.py`)

The reference `sample_preimage` feeds `t = (−c·F/q, c·f/q)` to
`ffsampling`. We replace this by its fractional part `t_frac = t − ⌊t⌉`
(in coefficient domain), computed as `q·t_frac = (−c·F, c·f) mod^± q` via
the NTT that is already in Falcon's stack. This:

- reduces `||t||_∞` at the root from `~n · γ_FG ≈ 2²⁰·⁸` (worst-case over
  adversarial c, with γ_FG = 3500) down to `≤ n/2 = 2⁸` (deterministic);
- preserves Falcon's output distribution (Lemma 10, `lem:sign`);
- is **KAT-preserving** byte-for-byte with shared randomness in the fxp
  pipeline (verified 1000/1000). In the float64 pipeline it diverges
  sporadically (~0/1000 to ~1/1000 depending on the secret key) because
  float t-construction has ~2⁻¹⁵ round-off that occasionally flips
  `floor(mu)` at integer boundaries; the fxp t-construction has ~2⁻⁴⁵
  round-off, six orders of magnitude below the failure threshold.

`sign_tweak.py` exposes two flags on `sign(...)`:
- `use_tweak: int ∈ {0, 1}`: `USE_TWEAK_STD=0` (no tweak),
  `USE_TWEAK_NTT=1` (Section-5.1 NTT-exact). Legacy `False`/`True` still
  work via `bool ⊂ int`.
- `use_fxp_ffsampling`: float reference (`False`) vs fxp pipeline
  (`True`). The fxp pipeline runs `_build_t_*_fxp` (NTT + `fft_fxp` +
  ·1/q) → `ffsampling_fxp` (with samplerz_fxp at the leaves)
  → `_ifft_round(z)` → `_reconstruct_s_int` (pure integer:
  s = (c, 0) − z'·B0 mod± q via NTT — since t·B0 = (c, 0) exactly, the
  signature needs no fixed-point at all). **No float on the
  per-signature critical path** (the only float→FxR conversions,
  `sk.sigma` and `sk.sigmin`, are cached on `sk` after the first
  signing).

## Format choices (`m` budgets)

Every `m` used in the pipeline is a fixed Falcon-512 constant defined —
with its derivation — in **`m_budgets.py`** (single source of truth).
`scripts/derive_m_budgets.py` re-derives each bound from the NTRUGen
thresholds and prints chosen vs derived (all budgets are proven;
`tests/test_m_budgets_derivation.py` asserts `chosen >= derived`).

The empirical evidence behind each bound, and the precision benchmarks
(FP vs FxP-63 vs FxP-127 vs mpmath-256), are reported in the paper.

## Testing

- `tests/test_fxtypes.py` — 52 unit tests covering FxR/FxC
  arithmetic, invariants, and edge cases.
- `tests/check_test_vectors.py` — ~60 end-to-end test vectors
  pinning the numerical behaviour of the FFT / ffLDL / division layers.
- `tests/smoke_test_e2e.py` — **integration smoke runner**.
  Runs `check_test_vectors.py` (and `experiments/sign_fxp_end_to_end.py`
  if present) in subprocesses and asserts that (a) every `verify_A..D` count hits
  `n_trials`, (b) no forbidden marker (`Squared norm too large`,
  `Traceback`, `AssertionError`) appears anywhere. This is the layer
  that catches placeholder/silent-pass regressions — e.g. an `or True`
  that masks a failed `verify`. Use `--fast` to skip the slow
  end-to-end case (~100 s).

## Running

```
# from the repo root:
python -m pytest tests/test_fxtypes.py     # unit (~0.6 s)
python tests/check_test_vectors.py         # numerical KAT (~0.6 s)
python tests/smoke_test_e2e.py --fast      # KAT only (~1 s)
python tests/smoke_test_e2e.py             # + sign+verify e2e (~110 s)
```
